# ACCEPTANCE_CRITERIA

## Phase 1
1. A creator can complete onboarding with role type, identity basics, and profile details.
2. A creator can access a dashboard with drafts, upcoming content, and quick actions.
3. A creator can create an event draft, edit it across steps, preview it, and publish it.
4. A creator can create a gallery draft, edit it across steps, preview it, and publish it.
5. Event and gallery content clearly display state: draft, scheduled, published, archived.
6. A creator can upload media assets and use them in events and galleries.
7. A creator can manage a public page with core sections enabled.
8. Published creator content appears in user-facing payload shapes.
9. The creator analytics screen shows at least summary metrics and top content.
10. Empty/loading/error states exist for all major creator surfaces.

## Phase 2
1. A creator can schedule event and gallery publication for a future time.
2. The public page supports section ordering and featured content configuration.
3. Analytics supports multiple date ranges and top-performing content views.
4. Form validation is field-specific and prevents invalid publish operations.
5. Stepper flows are resumable from saved drafts.
6. Venue owner-specific fields can be shown without degrading artist flow.
