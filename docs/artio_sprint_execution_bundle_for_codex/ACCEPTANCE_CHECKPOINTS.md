# Acceptance Checkpoints

## Sprint 1
- [ ] User can set a 2h or 24h reminder on an event
- [ ] User can delete a reminder
- [ ] User receives reminder notifications through supported channels
- [ ] User has a dedicated saved surface
- [ ] Notification preferences persist and affect behavior
- [ ] Feed shows improved rails and/or ranking reasons
- [ ] Followed creators influence feed or notifications
- [ ] Analytics are emitted for save/reminder flows

## Sprint 2
- [ ] Gallery exists as a first-class user concept
- [ ] Gallery list route exists
- [ ] Gallery detail route exists
- [ ] User can save a gallery
- [ ] Explore can surface both events and galleries
- [ ] Recommendations use at least some behavior signals
- [ ] Search supports the main user entities that actually exist in the repo

## Sprint 3
- [ ] Creator can create and publish a gallery
- [ ] Creator event flow has validation and preview
- [ ] Creator public page shows identity + content
- [ ] Scheduled publishing exists for at least one main content type
- [ ] Creator analytics shows actionable top-content metrics
- [ ] No admin/moderation surfaces were modified unintentionally
