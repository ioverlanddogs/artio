# Sprint 3 — Creator Completion

## Goal
Complete the creator experience needed to support a strong user-facing platform.

## Scope
Artists and venue owners only.
No admin/moderation work.

## Repository foundations already likely relevant
- /my dashboard
- artist profile editing
- venue management
- event creation/editing
- analytics
- publish controls
- image/media upload

## Tasks

### 1. Gallery publishing flow
Priority: Critical

Implement:
- create gallery
- upload/reorder media
- add captions/commentary
- preview
- publish
- edit after publish where supported

Acceptance:
- creator can publish a gallery as a first-class object

### 2. Event creation UX upgrade
Priority: High

Implement:
- clearer stepper or staged workflow
- field validation
- preview before publish
- better state labeling:
  - draft
  - published
  - archived
  - scheduled if added this sprint

Acceptance:
- event publishing is guided and confidence-building

### 3. Public creator page
Priority: High

Implement:
- creator mini-site/public page improvements
- sections:
  - hero
  - bio
  - upcoming events
  - galleries
- basic featured content controls if possible

Acceptance:
- creator identity is legible and compelling to users

### 4. Scheduled publishing
Priority: Medium

Implement:
- publish now vs schedule
- scheduler support
- state visibility for scheduled content

Acceptance:
- creators can schedule at least event publication, and galleries if supported

### 5. Creator analytics upgrade
Priority: Medium

Implement:
- top content
- views
- saves
- follows
- attendance intent where available
- date range support if feasible

Acceptance:
- creators can understand what content is performing

## Deliverables
- gallery publishing
- better event creation flow
- stronger creator public page
- scheduled publishing
- upgraded analytics

## Non-goals
- full CMS
- complex team workflows
- admin moderation features
