# Sprint Overview

## Strategic order

### Sprint 1 — Core User Loop Completion
Goal:
Make the app reliably useful for a basic user.

Loop:
Discover -> Save -> Remind -> Return

Why first:
This closes the most important retention loop and builds on existing recommendation, events, notification, and onboarding foundations already present in the repo.

### Sprint 2 — Gallery Product + Discovery Upgrade
Goal:
Turn artwork-centric browsing into a coherent gallery product and improve recommendation quality.

Why second:
The repo already supports artwork and recommendations, so the right next step is turning that into a better user-facing gallery experience.

### Sprint 3 — Creator Completion
Goal:
Complete creator publishing and identity features needed to support the user-side experience.

Why third:
Creator improvements should serve the user experience, not precede it. Public pages, gallery publishing, and scheduling are more valuable after user-side discovery is stronger.

## Success outcomes
- Users can save and set reminders on events
- Users can browse a coherent gallery product
- Recommendations feel more relevant
- Creators can publish galleries, manage identity, and understand performance
